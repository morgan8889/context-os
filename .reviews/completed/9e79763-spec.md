## Failed Acceptance Criteria

### T019 — `BriefingWorkflow.resume()` not implemented
`resume()` is a no-op. The checkpoint stored by `AsyncPostgresSaver` is never used to resume agent state after a human approval action. The interrupt-and-resume pattern required by Constitution Principle III (human governance gate) is non-functional. Callers in `api/inbox.py` do not call `resume()` at all despite `workflow_thread_id` being stored on `ApprovalItem`.

### T028 — `promote_dependency_edge` key mismatch
`src/context_os/graph/mutations.py:promote_dependency_edge` extracts `approved_content.get("from_node_id", "")` and `approved_content.get("to_node_id", "")`. The `MapperAgent.enqueue_proposals` node stores these as `from_initiative_id` and `to_initiative_id`. The Cypher MATCH will receive empty-string node IDs, match nothing, and create invalid orphaned edges (or no edges if the MATCH fails). Every approved dependency is silently lost from the canonical graph.

### T035 / T036 — `_compute_recall` always returns 1.0
`src/context_os/eval/mapper_eval.py`:
```python
ground_truth_count = sum(1 for r in records if r.ground_truth_exists)
if ground_truth_count == 0:
    return 0.0
proposed_true = sum(1 for r in records if r.ground_truth_exists)  # identical to denominator
return round(proposed_true / ground_truth_count, 4)              # always 1.0
```
The numerator should count records where both `r.ground_truth_exists` and `r.proposed` are true. The recall CI gate (`recall >= 0.50`) can never catch a mapper regression.
