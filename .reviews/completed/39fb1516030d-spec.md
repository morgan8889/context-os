## Critical Violations

### SPEC-1: `post_scope` skips mandatory "scope" step — state machine stuck (Confidence 90)

`post_scope` calls `svc.advance_step(session.id, "ingest", scope_data)` directly, skipping the "scope" step. The state machine enforces forward-by-one transitions, so this always raises `InvalidTransitionError` which is silently swallowed. `current_step` stays at "connect" forever.

Fix: advance "scope" first, then "ingest".

### SPEC-2: `POST /onboarding/scope` body diverges from OpenAPI contract (Confidence 92)

Contract specifies `jira_projects`, `github_repos`, `slack_channels` as separate arrays. Implementation uses `sources: list[str]`. FR-004 requires per-source granularity.

## Important Violations

### SPEC-3: Admin funnel missing `time_in_stage` field (Confidence 88)

FR-015 requires time-in-stage per row. `AdminFunnelRowResponse` has no such field.

### SPEC-4: Drop-off flag uses `updated_at` not step entry time (Confidence 85)

FR-015 and data-model SQL target `step_started_at[current_step]`; implementation uses `updated_at`.

### SPEC-5: Write endpoints not blocked during impersonation (Confidence 80)

FR-020 requires all writes blocked. `post_survey`, `post_scope`, `post_activation` use `get_current_tenant` not `check_not_impersonation`.
