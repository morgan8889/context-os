# Quality Review — 66507c8 (fix(web): Sigma render-program crash + Phase 3 visual verification enablement)

**Branch**: 3-cognition-surface
**Reviewer**: feature-dev:code-reviewer (agent a38105ad)
**Verdict**: changes-requested

The core Sigma `type` crash fix is correct, complete, and well-tested. Two
functional bugs (both pre-existing, surfaced by this review) and one security
gap should be addressed.

## Critical

### 1. `useGalaxyGraph` rebuilds the Graph on every render (confidence 90)
`web/src/views/galaxy/hooks/useGalaxyGraph.ts:142` — `buildGalaxyGraph(allNodes,
allEdges)` is unconditional and unmemoized. Each render returns a fresh `Graph`
reference, and `ForceLayout.tsx:38` (`useEffect([graph])`) then calls
`loadGraph(graph, true)` every render, restarting ForceAtlas2 from scratch.
Defeats the 5s convergence target (FR-002/SC-002) and tanks frame rate at 10k
nodes. The benchmark path (line 62-65) is correctly `useMemo`-wrapped; the live
path is not. (Pre-existing — the original inline `new Graph()` had the same
behavior — but worth fixing.)
**Fix**: wrap the node/edge collection + `buildGalaxyGraph` in `useMemo` keyed on
`nodesQuery.data` / `edgesQuery.data`.

### 2. `GalaxyActivating` passes unresolvable CSS strings as WebGL colors (confidence 95)
`web/src/views/galaxy/GalaxyActivating.tsx:42,55,73` — Sigma WebGL receives raw
`var(--color-node-*)` / `color-mix(...)` strings it cannot evaluate. Real nodes
render black; stubs miss placeholder-grey. Use the `getCSSVar()` pattern from
`GalaxyView.tsx:26-29` to resolve to computed values, applying opacity via the
nodeReducer. (Pre-existing; this commit only renamed `type`→`nodeType` on line 43.)

### 3. dev_router unauthenticated + unconditionally mounted (confidence 88)
`src/context_os/main.py:176` mounts `dev_router` with no gate; seed POST
endpoints overwrite the in-memory store with no auth/rate-limit. Docstring claims
`DEV_SEED_ENABLED` gating that does not exist. Least-privilege violation.
**Fix**: wrap the include in `if os.getenv("DEV_SEED_ENABLED","false").lower()=="true"`.

## Important

### 4. `ActivatingGraphLoader` effect dep is `nodes.length` not `nodes` (confidence 82)
`GalaxyActivating.tsx:67` — same-length-but-different node data won't reload the
graph. Count is ≤9 here so the perf shortcut buys nothing; prefer `[nodes]`
(TanStack Query arrays are reference-stable).

## Test coverage — strong
New `buildGalaxyGraph` tests assert `type` undefined + `nodeType`/`edgeType`
populated, missing-endpoint skip, and node count — directly covering the
regression. Hook-level test confirms the same end-to-end. `toInitiativeNode/Edge`,
`pointInPolygon`, `OverlayControls`, `NodeDetailPane` all well covered.
**Gap**: no test asserts that colors stored in graphology are resolved values
(not raw `var(...)`), so Issue 2 isn't caught (jsdom can't eval CSS vars + Sigma
is mocked).

  `toInitiativeNode` sets `x:0` (falsy) — dead check or wrong predicate.
- `TimeTravelBar.tsx:59`: GSAP `contextSafe` closure used in a suppressed-dep

- `viewState.ts:64-65`: `devState` binds `s` but ignores it for galaxy/topology.
