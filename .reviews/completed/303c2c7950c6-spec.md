## Issues

### S-1: FirstVisitCallout reads localStorage asynchronously (confidence 85)
**File**: `web/src/design-system/primitives/FirstVisitCallout.tsx`
**Spec**: Contract 1 — "Reads localStorage synchronously; sets dismissed state before first paint"
**Problem**: `useState(false)` + `useEffect` defers the read. Dismissed users get an extra
silent render; first-time users see the callout one frame later than specified.
**Fix**: `useState(() => localStorage.getItem(storageKey) !== 'true')` — remove first useEffect.

### S-2: aria-current={undefined} suppresses active-page accessibility (confidence 90)
**File**: `web/src/components/AppShell.tsx`
**Spec**: Contract 4 — `aria-current="page"` on active NavLink
**Problem**: Explicitly setting `aria-current={undefined}` overrides and removes the attribute
that React Router's NavLink would set automatically.
**Fix**: Remove `aria-current={undefined}` from the NavLink.

### S-3: Inbox view-orientation callout overlays card content (confidence 80)
**File**: `web/src/inbox/InboxView.tsx`
**Spec**: FR-001 — non-blocking orientation message
**Problem**: Fixed `position="bottom-left"` floats over approval cards in the list view,
unlike canvas views where the callout floats over whitespace. `position="bottom-center"`
would be more appropriate in the Inbox card-list context.
