## Async Correctness

The fix correctly targets `"context_os.ingestion.base.asyncio.sleep"` for the mock patch. This
is the correct module-level reference path — patching at the `asyncio.sleep` global would not
intercept the call inside the `ingestion.base` module because Python looks up `asyncio.sleep`
at call time via the module's own reference to the `asyncio` namespace.

The infinite-loop fix in the adapter (the commit message references this as "infinite-loop in
rate-limit fault tests") addresses a case where the retry logic would loop indefinitely if the
mock was not correctly intercepting the sleep. After the fix, the adapter correctly exits the
retry loop after the mocked second page returns `None` as the next cursor.
